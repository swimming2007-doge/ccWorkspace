## 1. 设计哲学

### 1.1 品牌定位

RideRecord 不仅仅是一个骑行记录工具，更是每一位骑手的**探险伙伴**。我们的设计语言传达：

| 核心价值 | 设计表达 |
|----------|----------|
| 🗺️ **探险精神** | 动感的线条、深邃的色彩、未知的神秘感 |
| ⚡ **科技智能** | 简洁的几何、流畅的动画、数据可视化 |
| 🎯 **个性表达** | 独特的视觉标识、差异化的功能展现 |
| 🌍 **户外友好** | 高对比度、大触控区、强光可读性 |

### 1.2 差异化视觉策略

与市面骑行应用对比：

| 竞品 | 视觉特点 | RideRecord 差异化 |
|------|----------|-------------------|
| Strava | 橙色活力、社交导向 | **探险主题色、智能感、地图优先** |
| 骑行西藏 | 传统绿色、功能堆叠 | **现代极简、数据可视化、沉浸体验** |
| 黑鸟单车 | 深色专业、复杂界面 | **平衡复杂与简洁、分层信息架构** |
| Keep骑行 | 健身风格、通用模板 | **骑行专属、探险视觉语言** |

### 1.3 设计原则

```
┌─────────────────────────────────────────────────────────────────┐
│                    RideRecord 设计原则                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   1. 🎯 焦点优先                                                 │

│                                                                  │
```

## 7. 动效规范

### 7.1 动效原则

| 原则 | 说明 |
|------|------|
| **有意义** | 动效传达状态变化，非装饰性 |
| **快速** | 交互反馈 < 100ms，过渡 < 300ms |
| **自然** | 使用缓动曲线，避免线性 |
| **克制** | 骑行中减少动效，专注数据 |
