# 骑行数据记录应用 - 软件需求规格说明书 (SRS)

**文档版本**: 1.0
**创建日期**: 2026-03-06
**项目代号**: RideRecord
**目标平台**: HarmonyOS (华为 Mate 70) + Huawei Watch 3 Pro

---

## 1. 引言

### 1.1 目的

本文档定义 RideRecord 骑行数据记录应用的软件需求规格，遵循 ISO/IEC/IEEE 29148 标准。本文档将作为项目开发、测试和验收的基础。

### 1.2 范围

RideRecord 是一款面向骑行爱好者的智能数据记录应用，运行于鸿蒙生态系统（华为 Mate 70 手机 + Huawei Watch 3 Pro 手表）。本应用的核心差异化价值在于：

1. **智能动作检测自动启停** - 通过检测用户骑行动作自动开始/暂停/结束记录，解决用户忘记手动操作的痛点
2. **探险导航模式** - 提供公路/探险/混合三种导航模式，支持冒险指数提示，满足探索需求
3. **全方位数据同步** - 支持云端存储、Web端查看、第三方平台同步

### 1.3 定义与缩略语

| 术语 | 定义 |
|------|------|
| HarmonyOS | 华为鸿蒙操作系统 |
| Huawei Watch 3 Pro | 华为智能手表，支持多种运动传感器 |
| GPX | GPS Exchange Format，GPS数据交换格式 |
| TCX | Training Center XML，训练数据格式 |
| Strava | 国际知名骑行/跑步社交平台 |
| 动作检测 | 通过传感器数据识别特定动作（上车、下车、踩踏等） |
| 探险模式 | 导航模式之一，包含土路、村道、山地车道等非公路路线 |
| 冒险指数 | 评估路线风险程度的指标（地形、交通、信号覆盖等） |

1. **硬件约束**

   - 手表：Huawei Watch 3 Pro

## 3. 功能需求

### 3.1 智能启停模块 (FR-001)

#### FR-001-01 动作检测自动开始

| 属性 | 描述 |
|------|------|
| 描述 | 系统检测到用户开始骑行动作时自动开始记录 |
| 输入 | 手表传感器数据（加速度计、陀螺仪） |
| 触发条件 | 检测到以下动作序列：<br>1. 用户佩戴手表<br>2. 检测到"上车"动作（腿部跨过车架）<br>3. 检测到开始踩踏（腿部周期性运动） |
| 输出 | 开始骑行记录，推送通知用户 |
| 响应时间 | 检测到动作后 3 秒内开始记录 |
| 异常处理 | 若误检测，用户可手动取消 |

#### FR-001-02 智能暂停与恢复

| 属性 | 描述 |
|------|------|
| 描述 | 检测到短暂停留时自动暂停记录 |
| 输入 | 手表传感器数据 + GPS速度数据 |
| 暂停条件 | 同时满足以下条件超过 5 秒：<br>1. 速度 < 2 km/h<br>2. 无踩踏动作<br>3. 心率下降 |
| 恢复条件 | 检测到踩踏动作或速度 > 5 km/h |
| 输出 | 暂停状态显示，数据标注为暂停时段 |
| 响应时间 | 暂停/恢复后 2 秒内更新状态 |

#### FR-001-03 自动结束骑行

| 属性 | 描述 |
|------|------|
| 描述 | 检测到结束骑行动作或长时间停留时，自动结束记录 |
| 输入 | 手表传感器数据 + 时间计时 |
| 触发条件 | 满足以下任一条件：<br>1. 检测到"下车"动作<br>2. 暂停状态持续超过 30 分钟 |
| 输出 | 弹出确认通知，等待用户确认后保存 |
| 响应时间 | 检测后立即推送确认通知 |
| 异常处理 | 用户可选择继续骑行或保存并结束 |

#### FR-001-04 误触保护机制

| 属性 | 描述 |
|------|------|
| 描述 | 防止误触发自动启停 |
| 实现 | 1. 开始记录时推送通知，用户可 10 秒内取消<br>2. 结束记录需用户确认<br>3. 动作检测算法需满足最小置信度阈值 |
| 可配置 | 用户可在设置中调整检测灵敏度 |

### 3.3 导航模块 (FR-003)

#### FR-003-01 公路导航模式

|------|------|
| 描述 | 提供基于公路的骑行导航，优先选择适合自行车的道路 |
| 路线偏好 | - 避开高速公路、快速路<br>- 优先选择有自行车道的道路<br>- 避开高拥堵路段 |
| 数据源 | 地图API + 实时交通数据 |
| 显示 | 路线、距离、预计时间、海拔变化图 |

#### FR-003-02 探险导航模式

| 属性 | 描述 |
|------|------|
| 描述 | 提供包含非公路路线的导航，支持土路、村道、山地车道 |

#### FR-003-04 导航语音提示

## 4. 非功能需求

### 4.1 性能需求 (NFR-001)

| ID | 描述 | 指标 |
|----|------|------|
| NFR-001-01 | 应用启动时间 | < 3 秒 |
| NFR-001-02 | 动作检测响应时间 | < 3 秒 |
| NFR-001-03 | GPS定位精度 | < 10 米 |
| NFR-001-04 | 数据同步延迟 | < 30 秒（良好网络） |
| NFR-001-05 | 导航路径计算时间 | < 5 秒 |
| NFR-001-06 | 内存占用（前台） | < 200 MB |
| NFR-001-07 | 电池消耗（骑行1小时） | < 15%（手机），< 10%（手表） |

### 4.2 可靠性需求 (NFR-002)

| ID | 描述 | 指标 |
|----|------|------|
| NFR-002-01 | 应用崩溃率 | < 0.1% |
| NFR-002-02 | 数据丢失率 | < 0.01% |
| NFR-002-03 | GPS信号丢失恢复 | 自动恢复，数据补全 |
| NFR-002-04 | 网络断开处理 | 本地缓存，恢复后同步 |
| NFR-002-05 | 手表断连处理 | 手机独立运行，重连后同步 |

### 4.3 安全性需求 (NFR-003)

| ID | 描述 |
|----|------|

## 8. 附录

### 8.1 动作检测算法初步方案

```
上车动作检测：
1. 手臂位置突变（从身体侧面到前方）
2. 腿部跨步动作（加速度计+陀螺仪）
3. 手臂稳定（握住车把）

踩踏动作检测：
1. 腿部周期性运动（频率 60-120 RPM）
2. 手臂轻微震动（车把传递）
3. 速度上升

下车动作检测：
1. 速度降至接近零
2. 腿部跨步动作（反向）
3. 手臂位置恢复
```

### 8.2 冒险指数计算模型

```
冒险指数 = Σ(因素权重 × 因素评分) / Σ权重

因素列表：
- 路面状况（权重 0.25）：铺装路=1, 砂石路=3, 崎岖路=5
- 坡度（权重 0.20）：<5%=1, 5-10%=2, 10-15%=3, >15%=5
- 交通流量（权重 0.15）：低=1, 中=2, 高=3
- 信号覆盖（权重 0.15）：良好=1, 一般=2, 差=4, 无=5
- 补给距离（权重 0.15）：<1km=1, 1-5km=2, 5-10km=3, >10km=5
- 天气条件（权重 0.10）：晴=1, 多云=2, 雨=4, 恶劣=5

冒险等级：
1-1.5：安全
1.5-2.5：低风险
2.5-3.5：中等风险
3.5-4.5：高风险
4.5-5：极高风险
```
