## 1. Test Scope

### 1.1 Applicable Categories

| Category | Applies | Rationale |
|----------|---------|-----------|
| Regression | ✅ Yes | All features must pass existing tests |
| Integration | ✅ Yes | 6 features with shared data flow (scraper → generator → poster) |
| E2E Scenarios | ✅ Yes | Multi-step workflow from paper crawl to blog post |
| Performance | ✅ Yes | NFR-001 specifies < 30s execution time |
| Security | ✅ Yes | Handles API tokens, external API calls |
| Accessibility | ❌ No | No UI features (CLI only) |
| Compatibility | ✅ Yes | Python 3.10+ requirement |
| Exploratory | ✅ Yes | Always applicable |

### 1.2 Test Environment

- **Platform**: Windows 11 / Linux
- **Runtime**: Python 3.10+
- **Network**: Public internet (ArXiv API, WordPress API)
- **Configuration**: config.yaml with test credentials

## 4. Risk-Based Prioritization

| Priority | Test Area | Risk | Mitigation |
|----------|-----------|------|------------|
| 1 | E2E Workflow | Core functionality broken | Full workflow test with real API |
| 2 | Integration (Scraper→Generator→Poster) | Data corruption between skills | Verify data integrity end-to-end |
| 3 | Performance (NFR-001) | Timeout under load | Measure actual execution time |
| 4 | Security | Token exposure, API errors | Verify token handling, error messages |
| 5 | Compatibility | Python version issues | Test on Python 3.10+ |

| Severity | Definition | Action |
|----------|------------|--------|
| **Critical** | System crash, data loss, API token exposure | BLOCK release |
| **Major** | Core workflow broken, NFR threshold failed | BLOCK release |
| **Minor** | Non-core affected, workaround exists | Document, may defer |
| **Cosmetic** | Log message issues, typo | Document, defer |
